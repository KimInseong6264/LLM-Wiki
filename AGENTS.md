# LLM Wiki - AGENTS.md

## 이 Wiki는 무엇인가?
Unity 게임 개발자의 개인 지식 베이스입니다.
공부한 내용, 개발 노하우, 재활용 코드를 마크다운으로 축적합니다.

## 폴더 구조
- `Knowledge/` : 지식 베이스 루트
  - `Unity/` : Unity 엔진 관련 노하우, 디자인 패턴, 팁
  - `CS_knowledge/` : 자료구조, 알고리즘, CS 이론, C# 언어 지식
  - `Code_snippets/` : 재활용 가능한 코드 모음 (C# 위주)
  - `Debug_log/` : 트러블슈팅 기록, 버그 해결 과정

## 파일 이름 규칙
파일명은 반드시 아래 형식을 따릅니다:
`[태그] 파일명.md`

### 새로운 태그 필요시
- 기존 태그에 맞지 않는 내용은 `[Unknown]` 태그로 파일 생성
- 반드시 유저에게 새로운 태그가 필요함을 알리고 태그 규칙 추가를 요청할 것
- 예: `[Unknown] 파일명.md` 생성 후 "새로운 태그 규칙이 필요합니다: [제안 태그명]" 알림

## Obsidian 태그 규칙
- 파일 상단에 파일명 태그와 키워드 태그를 함께 명시
- 파일명 태그: `#[폴더태그]` (예: #Git, #CSharp, #Unity)
- 키워드 태그: 내용의 핵심 개념들을 검색 및 분류에 유용한 주요 주제 위주로 3~5개 내외로 명시 (예: #GitFlow, #BranchingStrategy)
- `#Unknown` 태그는 Obsidian 태그로 사용 금지

## 문서 작성 및 지식 연결 전략
- **파일명 규칙**: `[태그] Title.md` 형식 준수 (파일명은 반드시 영문 PascalCase 사용)
- **문서 내부 제목**: 문서 첫 줄(H1)에는 국문 제목 사용 가능 (가독성 유지)
- **파일 명명 제한**: 파일명에 `#`, `!`, `?` 등 링크 구문과 충돌하는 특수문자 사용 금지 (`[` `]`는 허용)
- **태그(#) 활용**: 문서의 분류 및 카테고리화 용도로만 사용한다. (검색 효율 증대)
- **링크([[ ]]) 활용**: 
  - 문서 간의 물리적인 지식 연결(그래프 뷰 시각화)을 위해 적극적으로 사용한다.
  - 문서 작성 시 관련 키워드가 있으면 즉시 `[[링크]]`를 생성하고 `Related` 섹션에 추가한다.
  - 수정/보완 작업 시 관련 개념이 나오면 즉시 연결을 강화한다.
  - 상호 참조(Bidirectional)를 원칙으로 하여 지식이 파편화되지 않게 한다.
- **상시 최적화**: 고립된 문서(Orphan)가 발생하지 않도록 상시 그래프 뷰 연결을 강화하고, 주기적으로 최적화한다.
- **지식 확장**: 사용자로부터 제공받은 정보만을 정리하지 않는다. 관련 개념, 사례, 모범 사례 등을 적극적으로 조사하여 문서의 깊이를 더하고 견고하게 구성한다.

## 문서 작성 양식 및 규칙
문서 작성 시 반드시 `TEMPLATE.md`를 읽어 양식을 준수할 것. (작성 규칙은 본 AGENTS.md의 관리 규칙을 따름)

## 지식 문서 작성 스타일 (Knowledge Documentation Style)
지식 문서는 나중에 다시 봐도 외부 조사가 필요 없도록 깊이 있게 작성하며, 가독성을 위해 다음 형식을 준수한다:
1. **요약 우선**: 문서 최상단에 핵심 3줄 요약(TL;DR)을 배치한다.
2. **시각적 구조화**: 전략, 분류, 비교가 필요한 내용은 '표(Table)'를 사용하여 정리한다.
3. **아스키 아트(ASCII Art)**: 프로세스 흐름이나 구조는 텍스트 기반 아스키 아트로 명확하게 도식화한다. (외부 툴 렌더링 의존 방지)
4. **충분한 설명**: 핵심 키워드는 함축하지 말고, 개념과 이유를 명확히 설명하여 정보의 완결성을 갖춘다.
5. **구조 준수**: 소제목 아래에는 반드시 `---` 구분선을 삽입하여 가독성을 높인다.

## 코드 리팩토링 시 코딩 스타일 참고
- C# / Unity 코드 **리팩토링** 작업일 때만 `CODING_STYLE.md`를 읽고 규칙을 따를 것
- 신규 기능 구현, 버그 수정, 문서 작성, 일반 질문 응답 등 **리팩토링이 아닌 작업**에서는 해당 파일을 확인하지 않을 것
- Unity 스크립트 작성 시 `[SerializeField]` 위에는 반드시 `[Header("한글 필드명")]`을 명시하여 인스펙터 가독성을 높일 것.
