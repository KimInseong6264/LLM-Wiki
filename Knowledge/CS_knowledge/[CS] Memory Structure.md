# 1. 메모리 구조 개요

---

C# 프로그램이 실행되면 운영체제와 CLR(Common Language Runtime)에 의해 메모리 공간이 논리적으로 4개의 영역으로 분할됩니다. 각 영역은 데이터의 성격과 생명주기에 따라 다르게 관리됩니다.

# 2. 상세 메모리 영역

---

### 1) Code (Text) Segment
- **역할**: 프로그램이 실행할 기계어 코드(컴파일된 명령어)가 저장되는 영역입니다.
- **특징**: 프로그램 실행 중에 수정되면 안 되므로 **읽기 전용(Read-Only)**입니다.

### 2) Data (Static) Segment
- **역할**: 전역 변수, 정적(static) 변수, 상수 등이 저장됩니다.
- **특징**: 프로그램 시작 시점에 할당되며, 프로그램이 종료될 때까지 메모리에 유지됩니다.
- **관련 문서**: [[ [CSharp] Static ]]에서 정적(static) 필드의 동작 원리를 확인할 수 있습니다.

### 3) Stack Segment
- **역할**: 함수 호출, 매개변수, 지역 변수(Value Type)가 저장됩니다.
- **특징**:
    - **LIFO (Last In, First Out)** 구조로 빠르게 동작합니다.
    - 함수 호출 시 생성되고 종료 시 자동으로 해제되어 매우 효율적입니다.
    - 크기가 고정되어 있어 너무 깊은 재귀 호출 시 `StackOverflowException`이 발생합니다.

### 4) Heap Segment
- **역할**: 클래스 인스턴스(객체), 배열 등 참조 타입(Reference Type) 데이터가 저장됩니다.
- **특징**:
    - 프로그램 실행 중에 동적으로 크기가 변할 수 있습니다.
    - 가비지 컬렉터(GC)가 참조되지 않는 객체를 정리(해제)합니다.
    - Stack보다 할당 속도가 느리고 파편화 가능성이 있습니다.

# 3. 전체 메모리 구조 시각화

---

### 1) 계층 구조
메모리 주소의 높낮이에 따른 일반적인 계층 구조는 다음과 같습니다.

```text
[ 높은 주소 ]
+-------------------------+
|        Stack            | <-- 지역 변수, 메서드 프레임 (값 타입 저장)
+-------------------------+
|        Heap             | <-- 객체 인스턴스, 동적 할당 (참조 타입 저장)
+-------------------------+
|     Data (Static)       | <-- 정적 변수, 전역 변수
+-------------------------+
|      Code (Text)        | <-- 컴파일된 명령어 코드
+-------------------------+
[ 낮은 주소 ]
```

### 2) Value vs Reference 데이터 접근
Stack의 변수는 Heap의 실제 객체 주소를 가리키는 포인터(참조)를 가지고 있습니다.

```text
       [Stack Area]                   [Heap Area]
    +-----------------+             +---------------------+
    | Local Variable  |             |      Instance       |
    | (ref address)   | ----------> |     (Object Data)   |
    +-----------------+             +---------------------+
    | int value (10)  |             |   Array / Class     |
    +-----------------+             +---------------------+
```

---

# 4. 요약 및 주의사항

---

- **Value Type**: `int`, `float`, `struct` 등은 주로 **Stack**에 저장되어 성능이 빠릅니다.
- **Reference Type**: `class`, `string`, `Array` 등은 **Heap**에 저장되어 동적 관리가 가능하지만 가비지 컬렉션 부하를 고려해야 합니다.
- 효율적인 메모리 관리를 위해서는 짧은 생명주기를 가진 데이터는 Stack에, 동적이고 큰 데이터는 Heap에 배치하는 설계가 필요합니다.
