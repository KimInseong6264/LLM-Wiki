# 문맥 교환 (Context Switching)

- **Date**: 2026-06-07
- **Tags**: #CS #OperatingSystem #Process #Thread
- **Related**: [[[CS] Process]] [[[CS] Thread]] [[[CS] Interrupt]]]

---

# 1. 개요

---

### 1) 3줄 요약
- **정의**: CPU가 실행 프로세스를 P1에서 P2로 바꿀 때, 기존 상태를 저장하고 새 상태를 불러오는 과정.
- **오버헤드**: PCB 저장/복구 비용 + 캐시 미스(성능 저하의 주원인).
- **효율**: 프로세스보다 스레드 전환이 훨씬 가볍다.

# 2. 메커니즘

---

### 1) 흐름도
```text
    +-------------------------------------------+
    |           [프로세스 1 : Running]           |
    +--------------------+----------------------+
                         | (인터럽트 발생 / 상태 저장)
                         v
                [ OS / PCB ] (상태 교체)
                         | (상태 복구 / 다음 프로세스)
                         v
    +--------------------+----------------------+
    |           [프로세스 2 : Running]           |
    +--------------------+----------------------+
```

### 2) 핵심 구성 요소
- **PCB (Process Control Block)**: 프로세스의 상태 정보(레지스터, PID, 스케줄링 정보 등)를 담는 자료구조. CPU 레지스터 값을 임시로 보관하는 곳입니다.
- **PC (Program Counter)**: 다음에 실행할 명령어 주소. 문맥 교환 시 이 값을 PCB에 저장해두면, 나중에 다시 돌아왔을 때 중단된 지점부터 정확히 작업을 재개할 수 있습니다.

# 3. 효율성 및 오버헤드

---

### 1) 오버헤드 요인
- **연산 비용**: PCB 정보를 메모리와 CPU 레지스터 사이에서 옮기는 연산 자체가 CPU 자원을 소비합니다.
- **캐시 메모리 오염**: 프로세스 전환 시 기존 프로세스가 사용하던 캐시(데이터/명령어)가 무효화되어, 새로운 프로세스가 다시 캐시를 채우느라 성능이 일시적으로 저하됩니다.

### 2) 프로세스 vs 스레드 차이
- **프로세스 전환**: 독립적인 메모리 공간을 가지므로, 페이지 테이블 등 메모리 매핑 정보를 모두 갱신해야 하여 비용이 큽니다.
- **스레드 전환**: 같은 프로세스 내의 코드, 데이터, 힙 영역을 공유하므로 레지스터와 스택 정보만 변경하면 되어 훨씬 가볍고 빠릅니다.
